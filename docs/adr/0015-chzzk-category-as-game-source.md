# ADR-0015: 게임 정보원 = 치지직 category API (IGDB · 게임물관리위원회 기각)

- 상태: Accepted
- 날짜: 2026-07-18

## 맥락

게임 보드의 각 카드는 게임을 식별·표시할 정보(이름·이미지)가 필요하다. 정보원 후보는 둘이었다
— 범용 게임 DB인 **IGDB**(보조로 게임물관리위원회 API), 그리고 치지직 자체 **category API**.
이 사이트는 치지직 팬사이트라 방송 카테고리와 어휘가 정합하는 게 자연스럽고, 외부 계정·정규화
·인증 의존을 최소화하고 싶다.

치지직 category API 는 엔드포인트 하나다:

- `GET /open/v1/categories/search?query=&size=` — **`client_credentials` 인증 필수.**
- 반환 필드 4개: `categoryType`(`GAME`|`SPORTS`|`ETC`) · `categoryId`(안정 식별자) ·
  `categoryValue`(이름) · `posterImageUrl`(포스터 이미지).

## 결정

게임 정보는 **치지직 category API 하나**에서 온다. 반환 4필드만 쓰고, **IGDB·게임물관리위원회는
폐기**한다. 보드는 `categoryType='GAME'` 만 담는다.

이 API 가 `client_credentials`(서버 클라이언트 자격) 인증이라 **공개 보드 읽기 경로에서 매번
호출하지 않는다.** 게임을 추가·수정하는 시점에 4필드를 `games` 테이블에 **denormalize(스냅샷)**
하고, 공개 읽기는 그 스냅샷만 읽는다([ADR-0014](./0014-v1-data-model-schema.md)). 컴포저(추가
UI)는 이 검색 API 를 그대로 감싼 **카테고리 검색·선택** 흐름이 된다.

## 근거

- **치지직 팬사이트 → 동일 어휘.** 방송에서 쓰는 카테고리와 보드가 같은 식별자를 공유한다.
  별도 게임 DB(IGDB) 매핑·한국어 장르 화해·정규화 비용이 사라진다.
- **단일 소스 → 스키마 단순.** 장르 M:N·외부 식별자 컬럼이 불필요해진다. 카드 렌더에는 이름·
  포스터면 충분하다.
- **검색 UX 정합.** "게임 검색" 이 곧 치지직 카테고리 검색이라, 컴포저가 그 검색을 1:1 로 감싼다.
- **denormalize 로 공개 읽기가 외부에 무관.** `client_credentials` 를 공개 트래픽 경로에 노출하지
  않고, 치지직 API 의 가용성·레이트리밋에 읽기가 의존하지 않는다.

## 기각한 대안

- **IGDB(+ 게임물관리위원회 보조)** — 커버·평점·출시일·다중 장르 같은 리치 메타데이터를 얻지만,
  외부 계정·매 게임 매핑·한국어 화해·정규화 스키마 비용이 크다. v1 보드엔 이름·이미지면 충분하다.
- **매 읽기마다 category API 호출** — `client_credentials` 노출·레이트리밋·가용성 의존이 공개
  트래픽 경로에 얹힌다. 부적합.
- **`categoryId` 만 저장하고 값은 읽을 때 조회** — 위 문제에 더해 읽기 지연까지 생긴다.

## 결과

- (+) 스키마·의존이 최소가 되고 치지직 어휘와 정합한다.
- (+) 공개 읽기가 외부 API·인증과 무관해진다(스냅샷).
- (−) 치지직 GAME 카테고리에 없는 게임은 보드에 올릴 수 없다(현재 시드 8개 중 일부가 제외될 수
  있다 — 예: "마이 보이스 주"·"겟 투 워크").
- (−) 카테고리명·포스터가 치지직에서 바뀌면 스냅샷이 낡는다 → 수정 시 갱신하거나, 훗날 재동기
  경로가 필요할 수 있다.
- (−) 장르·평점 기반 리치 필터는 불가능하다(v1 범위 밖).
