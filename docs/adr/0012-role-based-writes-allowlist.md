# ADR-0012: allowlist channelId 기반 역할 쓰기

- 상태: Accepted
- 날짜: 2026-07-18

## 맥락

v1 정박점은 **공용 게임 보드 + 역할 기반 쓰기**다. 전원 치지직 로그인
([ADR-0006](./0006-chzzk-oauth-jwt-session.md))하되, 아무나 보드를 고치면 안 된다.
(반응/좋아요는 v1 범위 밖 — 로드맵.)

## 결정

- 읽기는 **공개**(로그인 불필요). 쓰기(game 추가·편집)는 **allowlist(channelId) 계정만**.
- 역할은 세션 `{channelId, role}` 에 담고, 쓰기 tRPC 뮤테이션에서 서버가 인가한다.
- 역할 부트스트랩: `SUPERADMIN_CHANNEL_ID` 가 최초 로그인 시 admin 으로 승격.
- **권한 상승 가드 + 감사 로그**: 비관리자는 스스로 role 을 못 바꾸고, 모든 역할 변경은
  기록한다.

## 근거

- channelId 는 치지직의 안정 식별자라 allowlist 의 키로 알맞다.
- 인가를 서버 뮤테이션에 두면 UI 분기(버튼 노출)는 편의일 뿐, 진짜 방어선은 서버가 된다 —
  클라이언트를 신뢰하지 않는다.
- 권한 상승 가드·감사 로그는 "역할 시스템"의 흔한 구멍(자기 승격)을 처음부터 막는다.

## 기각한 대안

- **누구나 쓰기** — 공용 보드가 스팸·훼손에 노출된다.
- **UI 로만 버튼 숨김** — 클라이언트 우회로 뚫린다. 서버 인가가 정본이어야 한다.
- **이메일/비번 계정** — 팬 커뮤니티 신원이 치지직에 있으므로 불필요한 비밀·마찰만 는다.

## 결과

- (+) 공개 읽기의 개방성과 쓰기의 통제를 동시에 얻는다.
- (+) 모든 쓰기가 Zod 입력 검증([ADR-0004](./0004-trpc-zod.md)) → 인가 → 감사 로그를 통과.
- (−) allowlist·역할 관리 UI 가 필요해진다(Phase 4). 비관리자 차단은 테스트로 못박는다.
