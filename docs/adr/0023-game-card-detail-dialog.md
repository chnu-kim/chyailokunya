# ADR-0023: 게임 보드 = 훑는 격자 + 카드 상세 다이얼로그

- 상태: Accepted
- 날짜: 2026-07-25
- 뒤집음: [ADR-0020](./0020-delete-confirm-dialog-replaces-undo-window.md) 의 "결과" 절이 기록한
  **수정·삭제의 카드 내 배치** — 그 버튼들은 이제 카드에 없다. 같은 문서의 **삭제 확인
  다이얼로그** 결정 자체는 그대로고, 오히려 이 변경이 그 백스톱을 덜 필요하게 만든다.
- 조정: [ADR-0015](./0015-chzzk-category-as-game-source.md) 의 게임 정보원(치지직 category)은
  그대로다. 바뀐 건 **그 검색에 없는 게임을 넣는 길을 언제 여는가**뿐이다.

## 맥락

사용자가 `/games` 를 열어 지적한 것이 여섯 가지였고, 그 중 넷이 한 뿌리에서 나왔다.

**격자가 카드마다 너무 많이 말한다.** 카드 한 장에 표지·이름·클리어 칩·플레이 날짜·수정
버튼·삭제 버튼이 서 있었다. 픽스처 8장이면 아이콘만 16개고, 그 대부분은 **지금 아무도 쓰지
않는다** — 보드를 여는 이유는 "뭘 했나 훑기"지 고치기가 아니다. 조작이 상시 보이는 근거는
견고했다(NN/g 가 hover 뒤에 숨기는 형태를 명시적으로 반대한다). 하지만 그 근거는 "조작을
숨기지 마라"이지 "조작을 모든 카드에 복제하라"가 아니다.

**컴포저는 한 화면에서 두 번 막았다.** 검색은 「검색」 버튼을 눌러야 나갔는데 그 단계가
화면에 안 보여서 사용자는 입력만 하고 기다렸다. 그리고 결과가 한 건이라도 나오면 직접 입력
경로가 통째로 닫혔다 — 치지직 카테고리에 아직 없는 신작·인디를 넣을 길이 사라진다. 그 규칙의
원래 근거는 "쉬운 길을 만들면 보드가 중복 표기로 갈라진다"였는데, 실제로 막힌 건 중복이 아니라
**정당한 사용자**였다.

**폼이 조용히 날아갔다.** 날짜를 고르려다 카드 밖을 스치면 배경 클릭으로 판정돼 입력이 통째로
사라졌다. 신호가 하나도 없었다.

**추가 폼이 클리어를 안 받았다.** 근거는 "추가하는 순간 이미 깬 게임은 드물다"였는데, 이 보드는
**이미 한 방송을 기록하는** 자리라 소급 입력이 정상 경로다. 플레이 날짜를 추가 폼에 되돌린
것과 정확히 같은 이유이고(이슈 #56), 그때도 문제는 왕복이었다.

## 결정

### 1. 격자는 표지·이름·클리어 칩까지만 싣는다

나머지(플레이한 날·클리어한 날)와 조작(수정·삭제)은 **카드를 눌러야 나오는 상세 다이얼로그**로
내린다. 클리어 칩만 앞면에 남긴다 — "이 게임을 깼나"는 훑는 눈이 던지는 질문이라 카드마다 열어
보게 하면 안 되고, 날짜와 달리 한 글자로 답이 된다.

**상세는 권한과 무관하게 열린다.** 거기 담긴 날짜는 공개 목록(`listGames`)이 이미 보낸 값이고,
앞면에서 뺀 정보를 권한 뒤로 숨기면 로그아웃 방문자는 전에 보이던 날짜를 잃는다. 권한이 가르는
건 조작뿐이다.

카드 전체가 히트 영역이 되지만 **카드를 통째로 `button` 으로 만들지는 않는다** — 그러면 그 안에
제목 요소를 못 둬(button 의 콘텐츠 모델) 보드가 제목 없는 이미지 더미가 된다. 제목 버튼의
`::after` 가 카드를 덮는다: 눌리는 것과 접근 이름이 같은 요소라 키보드·스크린리더에서도 하나다.

### 2. 수정·삭제는 상세를 **닫지 않고 그 위에** 띄운다

닫고 여는 쪽은 취소했을 때 돌아올 자리가 없어져 카드를 다시 찾아 눌러야 하고, 포커스도 갈 곳을
잃는다(방금 닫힌 모달의 버튼이 트리거다). 네이티브 `dialog` 스택에 얹으면 그 둘을 브라우저
기본 동작이 해결한다. 겹쳐 뜨는 다이얼로그는 스크림을 한 겹 더 깔지 않는다(40% 두 겹 = 64%).

### 3. 검색은 타이핑 debounce 로 자동 발사한다

「검색」 버튼을 없앤다. 늦게 온 응답을 버리는 규칙(`isStaleSearch`)은 그대로고, 자동 검색이
요청 빈도를 올리므로 오히려 더 중요해졌다. 버림의 흔적을 남기던 `staleDropped` 안내는 **삭제**
한다 — 버림을 일으킨 `queryChanged` 자체가 다음 검색을 예약하므로 "다시 검색해 주세요"가
거짓말이 됐다.

### 4. 직접 입력은 "결과 0건"이 아니라 "정확히 같은 이름이 없다"에서 연다

결과 목록의 **마지막 항목**으로 `+ '○○' 직접 추가` 한 줄을 붙이고, 이미 그 이름이 목록에 있으면
감춘다. 콤보박스의 creatable 패턴이 이 형태로 수렴해 있다(eBay 디자인 시스템·MUI Autocomplete ·
react-select 셋이 같은 규칙을 쓴다). 중복 우려는 여기서 갚는다 — 정확히 일치하는 정본 카테고리가
보이는데도 손으로 넣는 경로는 애초에 안 열린다. 0건일 때 열리던 옛 동작은 이 규칙의 특수한
경우로 그대로 남는다.

### 5. 미저장 입력이 있으면 셸의 닫기가 되묻는다

배경 클릭·Esc·모서리 X 는 **오조작으로 일어난다.** 그 셋만 가드하고, 본문의 「취소」·「뒤로」는
사용자가 고른 결정이라 그대로 닫는다. 무엇이 "고친 것"인지는 폼마다 다르므로 판정은 호출자가
한다(셸은 children 안을 못 본다).

### 6. 추가 폼이 클리어를 받는다

`addGameInput` 에 `cleared`·`clearedDate` 를 더한다. `default(false)` 라 안 보내는 호출자는 "안 깬
게임"으로 떨어진다 — 추가는 새 행을 만드는 연산이라 `updateGameInput` 의 "부분 patch 금지"(안 보냄과
지움이 구분 안 된다)가 성립하지 않는다. 지울 이전 상태가 없다.

## 대안

- **hover 로만 조작 노출** — ADR-0020 이전에 실제로 있던 형태고, 발견 가능성을 대가로 낸다.
  카드 클릭은 다르다: 조작을 숨기는 게 아니라 **한 단계 뒤로 옮기는** 것이고, 그 단계에 들어간
  사람은 이미 고칠 카드를 골랐다.
- **카드에 날짜만 남기고 조작만 내리기** — 사용자가 고른 안의 이웃이다. 격자는 조용해지지만
  카드마다 날짜 한 줄이 남아 "훑는 격자"라는 목적에서 절반만 간다. 사용자가 앞면 구성을
  세 안 중에서 직접 골랐다(표지 + 제목 + 클리어 칩).
- **직접 입력을 상시 노출** — 가장 빠르지만 검색보다 쉬운 길이 되어 옛 근거(중복 표기)가 그대로
  되살아난다. 검색을 한 번 돌린 뒤에만 여는 것으로 그 사이를 잡는다.
- **`window.confirm` 으로 미저장 확인** — ADR-0020 이 같은 이유로 기각했다(스타일 불가·접근
  이름과 포커스 복귀를 브라우저에 맡김). `GameDialog` 가 이미 있다.

## 결과

- (+) 격자가 사진과 이름만 말한다. 카드당 인터랙티브 요소가 2~3개에서 **1개**로 줄었다.
- (+) 카드 액션이 없어지며 그에 딸린 규칙이 통째로 사라졌다 — 광학 정렬(−13px), 카드 밖 유출
  검사, 파괴 액션 이격, `--trash`/`--pencil` 마스크. 상세엔 폭이 있어 글자 라벨을 그대로 쓴다.
- (+) 오식으로 삭제를 누를 확률이 내려간다. ADR-0020 이 "오식 가능성을 올리면서 백스톱을 없앤
  셈"이라고 적은 대가를 이 변경이 되돌린다 — 확인 다이얼로그는 그대로 두므로 백스톱은 둘이 된다.
- (+) 치지직 카테고리에 없는 게임을 **검색 결과가 있어도** 넣을 수 있다.
- (−) 날짜를 보려면 클릭이 한 번 든다. 훑기가 주 용도라는 판단에 건 값이고, 클리어 칩은 남겼다.
- (−) 자동 검색이 치지직 API 호출을 늘린다. 서버 프록시라 키는 안 새고(ADR-0015), debounce 350ms
  가 자모 단위 요청을 막는다.
- (−) 다이얼로그가 최대 세 겹까지 쌓인다(상세 → 수정 → 미저장 확인). 네이티브 `dialog` 의 top
  layer 스택이 감당하는 범위지만, 여기서 더 깊어지면 구조를 다시 본다.
