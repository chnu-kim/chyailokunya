# ADR-0021: e2e 로그인 상태 = 테스트 전용 키로 서명한 access 쿠키

- 상태: Accepted
- 날짜: 2026-07-20

## 맥락

[ADR-0008](./0008-vitest-workers-playwright.md) 이 세운 Playwright 스모크는 **비로그인 상태만**
돌았다. `e2e/` 에 세션을 만들 방법이 없어서다. 그 결과 로그인 상태 nav 의 자동 커버리지가 0 이
됐고(이슈 #23), 2026-07-19 헤더 재편에서 계정 영역을 크게 고치는 동안 스모크·시각 어디에도 그
상태가 한 번도 렌더되지 않았다. 당시엔 실컴포넌트가 내는 마크업을 DOM 에 주입해 CSS 만 확인했다.

주입의 한계는 이론이 아니라 실측으로 드러났다: 320px 로그인 브랜드 폭이 주입 기준 32.56px 인데
진짜 세션에선 **31.06px** 다. 주입 문자열은 컴포넌트가 구조를 바꿔도 낡은 채 초록으로 남는다.

제약이 두 개 겹쳐 있었다.

1. **세션은 서버가 만든다.** access 는 EdDSA 서명 JWT 이고 검증은 무상태다
   ([ADR-0017](./0017-self-session-eddsa-refresh-rotation.md)). 즉 테스트가 로그인 상태를 만들려면
   dev 서버가 검증에 쓰는 키를 **알아야** 한다.
2. **그 키를 dev 서버에 먹이는 경로가 사실상 하나뿐이다.** wrangler 는 로컬 시크릿을
   `.dev.vars.<환경>` → `.dev.vars` → `.env`/process.env 순으로 찾고 **앞이 하나라도 로드되면
   뒤를 아예 안 본다.** 대부분의 개발 머신엔 `.dev.vars` 가 있으므로 process.env 로는 주입할 수
   없다. `envFiles` 로 우회하는 길도 `@opennextjs/cloudflare` 가 그 옵션을 하드코딩해 덮어써 막혀
   있다.

## 결정

**e2e 는 테스트 전용 EdDSA 키쌍을 `.dev.vars.e2e` 에 생성하고, 그 키로 서명한 access 쿠키를
브라우저에 심어 로그인 상태를 만든다.** 치지직 OAuth 왕복은 타지 않는다.

- 키 생성: `scripts/e2e-dev-vars.mjs`. Playwright 의 **webServer 커맨드**가 dev 서버보다 먼저
  돌린다 — globalSetup 은 늦다(Playwright 는 webServer 플러그인을 globalSetup 보다 먼저 띄운다).
- 키 선택: `NEXT_DEV_WRANGLER_ENV=e2e` 로 wrangler 가 `.dev.vars` 대신 `.dev.vars.e2e` 를 읽게
  한다. `wrangler.jsonc` 에 `env.e2e` 섹션은 만들지 않는다.
- 서명: `e2e/session.ts` 가 **앱 코드(`signAccessToken`)를 그대로 호출**한다.
- 심기: `__Host-` 프리픽스 요건(Secure·Path=/·Domain 미지정)을 맞추려고 `https://localhost` url 로
  쿠키를 넣는다.

## 근거

- **진짜 경로가 돈다.** 미들웨어 access 검증 → `getServerActor` → `SiteNav` 까지 프로덕션 코드가
  그대로 실행된다. 서버는 이 세션을 진짜와 구별하지 못한다 — 구별할 근거가 서명뿐이고, 서명이
  진짜이기 때문이다.
- **프로덕션에 테스트용 분기를 만들지 않는다.** 앱 코드에 "테스트면 로그인 처리" 같은 문이
  생기면 그 문이 언젠가 배포로 샌다. 이 방식은 `src/` 를 한 줄도 건드리지 않는다(불변식 3 의
  방어선이 그대로 남는다).
- **클레임이 갈릴 수 없다.** 토큰 모양(`iss`/`aud`/`alg`/`kid`/클레임)을 테스트가 베껴 쓰지 않고
  앱 함수를 부르므로, 앱이 바뀌면 fixture 도 같이 바뀐다.
- **불변식 4 와 충돌하지 않는다.** 키는 실행 시 생성되는 일회성이고 `.dev.vars*` 가 gitignore 다.
  저장소에 개인키 모양의 문자열이 남지 않는다.

## 기각한 대안

- **마크업 주입(직전 상태)** — CSS 만 검증하고 컴포넌트가 그 마크업을 낸다는 보장이 없다. 실제로
  실측이 1.5px 어긋나 있었다. 이 ADR 이 대체하는 대상이다.
- **치지직 OAuth 를 실제로 태우기** — 외부 API 의존·비밀 필요·비결정성. e2e 가 잡으려는 건
  치지직이 아니라 우리 UI 다.
- **process.env 로 키 주입** — wrangler 탐색 순서상 `.dev.vars` 가 있으면 안 읽힌다(위 맥락 2).
  로컬에서만 조용히 안 먹는 최악의 실패 모양이라 기각.
- **`wrangler.jsonc` 에 `env.e2e` 섹션 추가** — 환경은 `d1_databases` 같은 키를 상속하지 않아
  DB 바인딩이 사라진다. 바인딩을 복제하면 프로덕션 설정이 갈라져 표류한다.
- **고정 키쌍을 저장소에 커밋** — 재생성 분기가 사라지는 장점은 있지만, 저장소에 개인키 모양의
  문자열을 두는 비용이 더 크다. 불변식 4 는 "무엇을 지키는 키인가"보다 먼저 읽힌다.
- **CI 워크플로에서만 키를 주입** — 로컬과 CI 의 경로가 갈라진다. 지금은 양쪽이 같은 스크립트를
  탄다(`.dev.vars` 없는 상태로 `CI=1` 재현 확인).

## 결과

- (+) 로그인 상태가 스모크(320·390·561·1280)와 시각 베이스라인 1장에서 실제로 렌더된다.
- (+) fixture 하나로 쓰기 UI·역할 분기 같은 후속 로그인 표면을 열 수 있다(`signIn` 의 overrides).
- (−) **`wrangler.jsonc` 에 `env.*` 섹션이 하나라도 생기면 e2e 의 dev 서버가 부팅에 실패한다.**
  환경명 `e2e` 가 정의돼 있지 않을 때 wrangler 가 이를 경고로 넘기는 건 `env` 키가 통째로 없을
  때뿐이고, 하나라도 있으면 에러로 승격된다. 경고 주석을 `wrangler.jsonc` 본문에 박아 뒀다.
- (−) `.dev.vars.e2e` 는 `.dev.vars` 를 **대체**한다(병합이 아니다). 그래서 이 환경엔
  `AUTH_URL`·`CHZZK_*` 가 없고, 로그아웃·쓰기 경로를 테스트에 붙이려면 그때 추가해야 한다.
- (−) 살아 있는 dev 서버는 부팅 때 읽은 키에 묶인다. `.dev.vars.e2e` 를 지우면 서버도 내려야
  한다(실패 시 안내 문구가 `e2e/session.ts` 의 `expectSignedIn` 에 있다).
