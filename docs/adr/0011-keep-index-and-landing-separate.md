# ADR-0011: index 와 landing 을 분리 유지 (병합 기각)

- 상태: Accepted
- 날짜: 2026-07-18

## 맥락

구 사이트는 `index.html`(카드 2장 런처)과 `landing.html`(실제 소개)의 정보 구조가
겹친다. "합칠까"는 구 저장소에서 이미 3/3 으로 기각된 열린 결정이었다. 마이그레이션이
이걸 임의로 뒤집으면 안 된다.

## 결정

두 화면을 **분리해서 이식**한다. Phase 2 에서 `index`(런처)와 `landing`(소개)을 각각
독립 라우트/컴포넌트로 옮긴다. 병합하지 않는다.

## 근거

- IA 병합은 사용자가 이미 기각했다 — 마이그레이션은 충실 이식이 먼저고, 리디자인은 나중이다.
- App Router 의 라우트 분리([ADR-0001](./0001-next-app-router.md))가 두 화면을 자연스럽게
  독립시킨다.
- 합칠지 여부는 이식이 끝난 뒤 사용자 결정 대상으로 남긴다 — 코드가 결정을 선점하지 않는다.

## 기각한 대안

- **이식하면서 하나로 합치기** — 사용자가 내린 결정을 에이전트가 뒤집는 것. 금지.

## 결과

- (+) 구 사이트의 확정된 IA 결정과 일관된다.
- (−) 두 화면의 일부 마크업이 겹친다 — 공유 크롬(nav·푸터·테마 토글)은 컴포넌트로 한 번만
  두어 중복을 흡수한다(구 사이트의 "크롬이 여러 HTML 에 복제"되던 문제의 해소).
